Comandos e Orientações

Git
Puxar atualizações da main
git pull origin main
Ver status das alterações
git status
Trocar para branch main
git checkout main
Voltar para branch de desenvolvimento
git checkout dev-pc1 git checkout dev-pc2
Backend (.NET)
Rodar o backend (da pasta raiz)
dotnet run --project backend/ZulexPharma.API
Rodar o backend (de dentro da pasta API)
cd backend\ZulexPharma.API dotnet run
Aplicar migrations no banco
cd backend\ZulexPharma.Infrastructure dotnet ef database update --startup-project ..\ZulexPharma.API
Matar processos dotnet travados
taskkill /f /im dotnet.exe PowerShell: Stop-Process -Name dotnet -Force
Sincronismo
Ativar sync no PC1 (Filial 1)
$env:SYNC_PROFILE = "pc1" dotnet run
Ativar sync no PC2 (Filial 2)
$env:SYNC_PROFILE = "pc2" dotnet run
Limpar controle de sync (SQL)
DELETE FROM "SyncControles";
Central (Railway)
O Railway sobe automaticamente ao dar push na main. Ele NÃO roda o sync — só recebe e envia dados.
Intervalo do sync
30 segundos (configurável em appsettings.pcX.json)
Frontend (Angular)
Rodar o frontend
cd frontend ng serve
Acessar no navegador
http://localhost:4200
Build de produção
ng build --configuration=production
